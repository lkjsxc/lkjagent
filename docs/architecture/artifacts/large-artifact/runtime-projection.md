# Runtime Projection

## Purpose

Define the compact artifact facts projected from the store into resolver,
prompt, CLI, watch, task inspection, and completion gates.

## Fact Shape

```text
artifact_root=...
artifact_profile=...
plan_status=...
atom_total=...
atom_ready=...
atom_missing=...
next_atom=...
next_path=...
active_contract=...
measured_total=...
accepted_floor=...
assembly_pending=...
readiness=...
completion_blockers=...
```

## Resolver Use

The resolver chooses `artifact.plan` when no plan exists, `artifact.next` when a
next atom exists and no active contract exists, `fs.batch_write` when an active
contract exists, `artifact.audit` when writes or assembly need measurement, and
completion only when readiness is ready and graph evidence is complete.

## Prompt Card

A content-write prompt card names the active contract id, atom id, exact path,
count floor, target count, byte budget, required sections, continuity digest,
weak classes, and one line-protocol `fs.batch_write` example. The card excludes
unrelated transcript text and excludes prewritten prose.
