# Document Structure Redesign

## Purpose

This file owns the semantic scaffold and audit redesign for documentation
trees generated by lkjagent tools.

## Contract

- Ordinary documentation uses semantic directories and filenames.
- Every generated directory has a README.md local index.
- Generated roots include a `catalog.toml` manifest.
- Sequence-only filenames are forbidden outside a deliberate ordered archive.
- Generated leaves do not repeat a universal contract, hooks, failure, and status body.
- Structure-only pages are explicit repair targets and do not satisfy artifact readiness.
- Multi-topic requests become request, relation, and topic pages instead of one long slug.

## Implementation Hooks

- Source: `crates/lkjagent-tools/src/doc/body.rs`
- Source: `crates/lkjagent-tools/src/doc/profile.rs`
- Source: `crates/lkjagent-tools/src/doc/names.rs`
- Tests: `crates/lkjagent-tools/tests/doc_boilerplate.rs`
- Tests: `crates/lkjagent-tools/tests/doc_content_readiness.rs`
- Verification: `cargo test -p lkjagent-tools --test doc_boilerplate`
- Verification: `cargo test -p lkjagent-tools --test doc_content_readiness`
- Verification: `cargo run -p lkjagent-xtask -- check-docs`
- Verification: `cargo run -p lkjagent-xtask -- check-lines`

## Failure Modes

- `doc.scaffold` emits part-001.md, part-002.md, or equivalent names.
- `doc.scaffold` emits repeated generic leaf bodies across unrelated paths.
- `artifact.next` proposes filler prose instead of path-specific evidence work.
- README indexes omit local Markdown files or child directories.
- Audit passes a root that lacks the catalog metadata or semantic coverage.
- Audit reports topology as readiness for structure-only generated pages.
