# Commonality Discovery

## Purpose

Commonality discovery owns extraction of repeated concepts into one canonical
page. It reduces duplication without hiding evidence or weakening links.

## Extraction Candidates

- repeated hard-state explanations move to [../state/hard-state.md](../state/hard-state.md).
- repeated output grammar moves to [../model-interface/contract.md](../model-interface/contract.md).
- repeated Docker gate text links to [../operations/verification.md](../operations/verification.md).
- repeated no-op maintenance summaries move to [no-op-suppression.md](no-op-suppression.md).

## Refactor Rule

A refactor moves one coherent cluster, updates backlinks and README tables,
runs the owning audit, and records the result before selecting another cluster.
