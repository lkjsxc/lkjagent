# Operations

## Purpose

This directory owns how the project is verified and how the harness is run:
the gate commands and their quiet output contract, the docker compose
service design, and the runtime configuration. Verification rules here bind
every change through [../repository/workflow.md](../repository/workflow.md).

## Table of Contents

- [verification.md](verification.md): gates, the quiet contract, and the interim checks.
- [compose.md](compose.md): the docker compose services and their guardrails.
- [running.md](running.md): configuration keys and day-to-day operation.
