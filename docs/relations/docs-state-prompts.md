# Docs State Prompts

## Purpose

This relation page connects documentation growth, hard state, weighted tracks,
prompt frames, audits, and completion gates.

## Relations

- DocumentationContract depends on objective normalization and topic roles.
- Hard state blocks mutation until the active documentation stage allows it.
- Weighted tracks add guards for mock content, model naming, and connectivity.
- Prompt frames include the growth stage, missing evidence, and next action.
- Semantic audits update evidence and unblock completion only when fresh.
