# Diagnostic Output

## Purpose

This directory holds generated-output evidence that is useful for testing but
must not be mistaken for completed owner work.

## Table of Contents

- [known-failures.md](known-failures.md): failure classes represented by this
  evidence tree.
- [structured-output-scaffold/](structured-output-scaffold/README.md): failed
  scaffold record for the previous structured-output workspace.

## Contract

A child directory here is a diagnostic fixture or evidence record. It remains a
failed output until an audit names concrete content, the current artifact id,
and the passing readiness profile.
